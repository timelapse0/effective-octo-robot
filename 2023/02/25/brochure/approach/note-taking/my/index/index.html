网络笔记

纸质笔记终究在电子笔记的比拼下落了下风。但碍于学生记笔记的条件有限——不多数学校并不允许带电子产品上课，将纸上的笔记转换成电子文本也十分耗费宝贵的高中时间，所以后面再介绍纸质笔记。

最初纠结了一圈，我没有选择纸质笔记，我最终选择的是网页可浏览。

电子版网络笔记的优势毫无疑问有大量优点——方便修改和转移，排版和字体整齐美观[1]，方便向大众展示等。

最初，还有很多当时身为小白的我没有意识到的优势，比如运用 CSS 可以很方便地控制整体样式。

简单谈谈前面提到的几个因素：

  • 方便编写：对于电子笔记，要能支持多种语法。
  • 方便即时阅读
  • 方便修改和转移：我选择的是打字；不是手写。

从原生 HTML 到博客框架协助

私人笔记的选择有 OneNote、Notion 等笔记软件、腾讯文档等网上编辑软件，wiki farm、TiddlyWiki 等 wiki 系统，Github Pages等页面托管平台等等，其中 Github 提供的 Github Pages 可以展示没有后台服务的静态页面，高度（完全）的自定义页面，完全免费，国外服务器也不需要备案，很适合用作个人网页展示。你所见的 Github Pages 的页面渲染前的原始文件是 HTML 格式的，需附加的字母和符号繁多，不适合于日常做笔记。（。）

那些博客生成器就为我们提供了很好的选择，可以将 Markdown 文件 转化成 HTML 文件，再由 Github 部署。Hexo 简洁快速，网上教程和记录众多，我就选择了它。

  • 无需关注具体原理，只用挑选心仪的 Hexo 博客主题，然后对主题文件进行少量的自定义修改， Hexo + Github Pages 就能把你所写的 Markdown 文件转化成你自己的博客网站。我们还可以添加插件，丰富功能，比如全站搜索。

无论是个人学习笔记记录，还是个人知识库搭建，都是适用的。

Markdown 编写，主题文件修改，Github 部署出版，个人的学习网站就简洁方便地搭建完成了~

横向对比

我使用过 TiddlyWiki、坚果云、腾讯文档、WikiFarm 等多种网络编辑软件/工具，包括某小厂的笔记网站，

以下基于我的认知

对象 语言支持 是否方便网络浏览 是否适合多人编辑 打开速度 样式自定义程度 优势/劣势
坚果云 / 1 0 快 不佳 /
腾讯文档等 / 1 1 快 不佳
TiddlyWiki 可通过插件添加 1 0 慢 佳 学习成本高
WikiFarm / 1 1 以具体网站而定 不佳 wiki 链接
Obsidian Markdown，HTML，Mermaid，LaTex 0 0 快 佳 wiki 链接，所见即所得

谈谈纸质笔记

  1. 我认为，手写体有很大的不稳定性（特指字体上），即使认真书写，随着审美的改变，字体样式与美观度的微笑改变也可能引起本人的不适感（尤其对我而言），为了防止自己写的东西后面自己都看不下去了的情况发生（。），就可以让网络笔记作为解决途径。 ↩
